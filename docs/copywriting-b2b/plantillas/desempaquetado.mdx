## Qué es este documento

Esta es la plantilla estándar para producir un documento de desempaquetado comercial. Se usa una vez por vertical de negocio de cada cliente.

El desempaquetado toma el material crudo (temario, reuniones, brief) y lo organiza en bloques listos para que el copywriter escriba sin tener que buscar datos en múltiples documentos.

---

## Instrucciones

1. Copiar esta plantilla a `{proyecto}/docs/copy/desempaquetado-{vertical}-{empresa}.md`
2. Para cada página o sección del sitio, crear un bloque siguiendo la estructura de abajo
3. Llenar cada campo. Si no hay datos disponibles, marcar como **[HUECO]** para que el equipo sepa qué falta
4. El desempaquetado se hace ANTES de escribir copy — nunca después

---

## Estructura por página/sección

Repetir este bloque para cada página del sitio que necesite copy:

```markdown
### PÁGINA: [nombre de la página y ruta, ej: "Diferencia Nelson — /nelson/diferencia-nelson"]

**Bloques fuente:** [listar los IDs o nombres de los bloques del temario, reuniones o brief que alimentan esta página]

**Material crudo relevante:**
[Extractos concretos del temario, reuniones o brief que aplican a esta página. Copiar los datos directamente — no interpretar ni resumir. Incluir citas textuales de reuniones si son relevantes.]

**Ángulo comercial:**
[1-3 oraciones que resumen POR QUÉ este contenido le importa al visitante. No es el copy — es la dirección estratégica que debe tomar el copy.]

**Temas comerciales que alimenta:** [listar cuáles de los 10 temas comerciales (o los que tenga el proyecto) se alimentan con este material, ej: #1 World-Class, #6 Trusted Partner]

**Datos duros para el copy:**
[Lista de bullets con todos los números, nombres propios, fechas, certificaciones y métricas disponibles para esta página. Estos son los datos que el copywriter va a usar directamente.]
- [dato 1]
- [dato 2]
- [dato 3]

**Huecos de información:**
[¿Qué datos nos faltan para escribir esta página completa? ¿A quién hay que preguntarle?]
- [HUECO]: [descripción de lo que falta y quién podría tenerlo]
```

## Notas

- Un desempaquetado puede cubrir múltiples páginas del sitio. No es necesario crear un archivo separado por página.
- Los datos del desempaquetado deben ser textuales y concretos — no interpretaciones. El copywriter interpreta; el desempaquetado organiza.
- Si durante el desempaquetado se identifican huecos críticos (datos que el copy necesita y no existen), estos se escalan al cliente antes de escribir.
